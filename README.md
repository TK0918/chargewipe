# 拒付预警系统 - 原型页面

## 📁 目录结构

```
Chargewipe/
├── admin/                    # 运营端（中文版）
│   ├── common.css           # 运营端通用样式（Element Plus风格）
│   ├── login.html           # 登录页面
│   ├── dashboard.html      # 数据看板（单页应用）
│   ├── customers.html       # 客户管理（单页应用，包含所有子页面）
│   ├── providers.html       # 服务商管理（单页应用，包含所有子页面）
│   └── alerts.html          # 预警明细（单页应用，包含所有子页面）
│
└── client/                   # 客户端（英文版）
    ├── common.css           # 客户端通用样式（Ant Design风格）
    ├── login.html           # 登录页面（包含忘记密码功能）
    ├── authorizations.html  # 授权管理（单页应用，包含所有子页面）
    ├── alerts.html          # 预警管理（单页应用，包含所有子页面）
    └── wallet.html          # 钱包管理（单页应用，包含所有子页面）
```

## 🎯 页面结构说明

### 运营端（5个HTML文件）

1. **login.html** - 登录页面
   - 邮箱+密码登录
   - 登录失败5次锁定15分钟提示

2. **dashboard.html** - 数据看板
   - 核心指标卡片（总预警数、总成本、总收入、毛利润、毛利率）
   - 每日趋势图
   - 每月汇总表
   - 按服务商维度统计表
   - 按客户维度统计表

3. **customers.html** - 客户管理（单页应用）
   - **客户列表页**：搜索、筛选、分页、操作按钮
   - **新增客户页**：基本信息、余额设置、定价设置（固定/阶梯）
   - **客户详情页**：基本信息、授权列表、预警统计
   - **编辑客户页**：修改基本信息、账号状态、余额阈值
   - **客户定价设置页**：固定价格/阶梯价格设置
   - **客户余额管理页**：调整余额、设置阈值、余额变动记录

4. **providers.html** - 服务商管理（单页应用）
   - **服务商列表页**：搜索、筛选、分页、操作按钮
   - **新增服务商页**：基本信息、支持模式、数据获取方式
   - **服务商详情页**：基本信息、预警统计
   - **编辑服务商页**：修改基本信息、支持模式、数据获取方式、状态
   - **API配置页**：推送模式配置（Webhook URL、Token）、轮询模式配置（Endpoint、Key、Secret）、通用接口配置
   - **成本价格设置页**：固定价格/阶梯价格设置

5. **alerts.html** - 预警明细（单页应用）
   - **预警列表页**：搜索、筛选、分页、批量导出
   - **预警详情页**：预警信息、扣费信息、关联信息、反馈信息（ETHOCA）
   - **预警分配页**：手动分配未匹配的预警给客户

### 客户端（4个HTML文件）

1. **login.html** - 登录页面
   - 邮箱+密码登录
   - 忘记密码功能（弹窗形式）
   - 登录失败5次锁定15分钟提示

2. **authorizations.html** - 授权管理（单页应用）
   - **授权列表页**：搜索、筛选、状态标识、操作按钮
   - **新增授权页**：Descriptor、授权模式（ETHOCA/RDR）、BIN/CAID（RDR必填）、通知邮箱、服务商选择
   - **授权详情页**：基本信息、状态历史
   - **编辑邮箱页**：修改通知邮箱（仅Active状态可编辑）

3. **alerts.html** - 预警管理（单页应用）
   - **预警列表页**：搜索、筛选、状态标识、操作按钮
   - **预警详情页**：预警信息、扣费信息、反馈信息（已反馈的ETHOCA）
   - **ETHOCA反馈页**：退款单号、退款金额、退款时间（仅Pending Feedback状态可反馈）

4. **wallet.html** - 钱包管理（单页应用）
   - **交易明细Tab**：交易列表、搜索、筛选、分页
   - **在线充值Tab**：充值金额、支付方式选择
   - **线下转账Tab**：转账金额、转账日期、上传凭证、备注
   - **转账审核状态Tab**：转账记录列表、状态查看、凭证查看

## 🚀 使用方法

1. **直接打开HTML文件**
   - 在浏览器中直接打开对应的HTML文件即可查看
   - 页面内的子页面通过JavaScript切换显示，无需跳转

2. **使用本地服务器（推荐）**
   ```bash
   # 使用Python启动本地服务器
   cd Chargewipe
   python -m http.server 8000
   
   # 然后访问
   # 运营端: http://localhost:8000/admin/login.html
   # 客户端: http://localhost:8000/client/login.html
   ```

3. **页面导航**
   - 运营端和客户端都有侧边栏/导航栏，可以点击切换主模块
   - 每个主模块内的子页面通过页面内的按钮和链接切换
   - 使用面包屑导航可以返回上级页面

## 📝 功能说明

### 运营端核心功能
- **数据看板**: 展示总预警数、总成本、总收入、毛利润、毛利率等核心指标，支持多维度统计
- **客户管理**: 客户CRUD、价格设置（固定价格/阶梯价格）、余额管理、余额预警阈值设置
- **服务商管理**: 服务商CRUD、API配置（推送/轮询）、成本价格设置、测试连接
- **预警明细**: 预警查询、手动分配、状态管理、批量导出

### 客户端核心功能
- **授权管理**: 新增授权（支持ETHOCA/RDR）、查看授权状态、编辑邮箱、停用/恢复
- **预警管理**: 查看预警列表、ETHOCA预警反馈、RDR预警查看
- **钱包管理**: 在线充值、线下转账、交易明细查询、转账审核状态查看

## 🎯 设计要点

1. **单页应用设计**: 每个主模块一个HTML文件，所有子页面通过JavaScript切换显示
2. **响应式设计**: 所有页面都支持响应式布局，适配不同屏幕尺寸
3. **表单验证**: 关键表单字段都有必填标识和基础验证
4. **状态标识**: 使用不同颜色的标签（Tag）标识各种状态
5. **交互反馈**: 按钮、链接都有hover效果，提升用户体验
6. **数据展示**: 表格、卡片、统计图表等多种数据展示方式
7. **面包屑导航**: 子页面都有面包屑导航，方便返回上级页面

## 📌 注意事项

1. **API集成**: 当前页面中的表单提交都是模拟的，实际使用时需要连接后端API
2. **数据展示**: 页面中的数据都是示例数据，实际使用时需要从后端获取
3. **页面切换**: 子页面切换通过JavaScript实现，所有功能都在同一个HTML文件内
4. **样式定制**: 可以根据实际需求调整CSS样式文件
